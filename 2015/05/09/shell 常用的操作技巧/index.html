shell 常用的操作技巧

重定向操作

#0 stdin标准输入
#1 stdout标准输出   
#2 stderr标准错误

#文件打开模式：
#> 等同于1> 截断模式
#>>等同于1>> 追加模式
#<用于从文件中读取至stdin 只读模式

#stdout不会有任何输出，因为错误已经重定向到out.txt中了
ls + 2> out.txt
#可以将stderr单独重定向到一个文件，将stdout重定向到另一个文件：
cmd 2>stderr.txt 1>stdout.txt
#将stderr转换成stdout，使得stderr和stdout都被重定向到同一个文件：
cmd 2>&1 output.txt
#或者
cmd &> output.txt

#将stderr输出丢弃
some_command 2> /dev/null

#将文件重定向到命令
cmd < file

#向log文件中写入头部数据
#!/bin/bash
cat <<EOF>log..tt
LOG FILE HEADER
this is a test log file
EOF

#cmd >a 2>a 和 cmd >a 2>&1 为什么不同？
#cmd >a 2>a ：stdout和stderr都直接送往文件a ，a文件会被打开两遍 ，由此导致stdout和stderr互相覆盖。
#cmd >a 2>&1 ：stdout直接送往文件a ，stderr是继承了FD1的管道之后，再被送往文件a 。a文件只被打开一遍，就是FD1将其打开
#他们的不同点在于：
#cmd >a 2>a 相当于使用了FD1、FD2两个互相竞争使用文件 a 的管道； 
#而cmd >a 2>&1 只使用了一个管道FD1， 但已经包括了stdout和stderr。
#从IO效率上来讲，cmd >a 2>&1的效率更高。

History

|命令 | 说明  |
| ------------ | ------------ |
|^foo| 删除上一条命令中的foo|
|^foo^bar|将上一条命令的第一个foo改为bar|
|!!|重新执行上一条命令|
|!N|重新执行第N条命令|
|!-N|重新执行倒数第N条命令|
|!string|重新执行以字符串开头的命令|
|!?string?|重新执行包含字符串的命令|
| !$|上一条命令的最后一个参数|
|!^|上一条命令的第一个参数|
|!cmd:n|上一条命令的第n个参数|
|!*|上一条命令的所有参数|
|!*:x|x表示修饰符|

移动类

移到命令行首：`Ctrl + a(head)`
移到命令行尾：`Ctrl + e(nd) `
在命令行首和光标之间位置切换：`Ctrl + x + x `
右向按字符前移：`Ctrl + f(orward) `
左向按字符后移：`Ctrl + b(ackward) `

删除类

注释当前行：`Alt + # `
从光标处删除至命令行首：`Ctrl + u `
从光标处删除至命令行尾：`Ctrl + k `
从光标处删除至字首(以空格作为word边界)：`Ctrl + w `
从光标处删除至字首(以空格，斜线，点号作为word边界)：`ESC + Backspace `
从光标处删除至字尾：`Alt + d `
删除光标处的字符：`Ctrl + d `
删除光标前的字符：`Ctrl + h `
粘贴至光标后：`Ctrl + y `
重新调用前一个命令中的参数,非常有用!：`Esc+.`

大小写快速转换

从光标处更改为首字母大写的单词：`Alt + c `
从光标处更改为全部大写的单词：`Alt + u `
从光标处更改为全部小写的单词：`Alt + l `

顺序快速调换

交换光标处和之前的单词：`Alt + t `
交换光标处和之前的字符：`Ctrl + t `

撤销修改

C-/ 或 C-_
撤销当前行的所有内容：`Alt+r`

清屏

清屏：`Ctrl + l`
clear

查找历史命令

逆向搜索历史命令行和当前命令行：`Ctrl + r`
TIPS　: 在当前行中搜索,可以快速定位到需要修改的字符:Ｐ
终止搜索并还原原始命令行：`Ctrl + g`
终止搜索并停留在查找的位置，直接使用ESC也可以实现该功能：`Ctrl + j`
历史中的上一条命令：`Ctrl + p`
历史中的下一条命令：`Ctrl + n`
非增量搜索：`alt + n\p`
执行当前命令，并选择上一条命令：`Ctrl + o`
